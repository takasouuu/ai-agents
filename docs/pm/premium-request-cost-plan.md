# Premium Request 消費計画（4ヶ月）

## 1. 目的
- Premium Request の消費量と利用コストを事前に見積もり、予算超過を防ぐ。
- 週次で実績を確認し、工程別に利用配分を調整する。

## 2. 前提値（入力する）
- 単価（1 Premium Request あたり）: `X` 円
- 月間予算上限: `Y` 円
- 4ヶ月総予算上限: `Y * 4` 円
- 予備率（バッファ）: `10%`

## 3. 見積式
- 週次想定コスト = `週次想定リクエスト数 * 単価`
- 月次想定コスト = `月次想定リクエスト数 * 単価`
- 4ヶ月想定コスト = `合計想定リクエスト数 * 単価`
- 予備込み4ヶ月予算 = `4ヶ月想定コスト * 1.10`

## 4. 工程別の初期配分（例）
- 立ち上げ（W1-W2）: 10%
- 基本設計（W3-W5）: 20%
- 詳細設計（W6-W8）: 20%
- 実装+単体（W9-W13）: 30%
- 内部結合（W14-W16）: 20%

## 5. ロール別の初期配分（例）
- PL/PMO: 15%
- 設計担当: 30%
- 実装担当: 25%
- QA/テスト担当: 20%
- 品質/セキュリティ担当: 10%

## 6. しきい値と制御ルール
- `Green`: 予算消化率 80% 未満
- `Yellow`: 予算消化率 80%以上 95%未満（翌週配分調整）
- `Red`: 予算消化率 95%以上（高優先度タスク以外を制限）

### 制御アクション
- Yellow時:
  - 低優先度の試行回数を削減する
  - テンプレ化済みタスクは再利用優先に切替する
- Red時:
  - 人レビュー直前の最終確認など高価値用途へ限定する
  - プロンプト標準化で再実行を抑制する

## 7. 週次レビュー項目
- 予定 vs 実績（件数・コスト）
- 工程別消費の偏り
- 再実行率（同一タスクのやり直し率）
- コスト効率（1成果物あたりリクエスト数）
- 翌週の配分変更案

## 8. 可視化指標（最低限）
- 累積消費件数
- 累積コスト
- 予算消化率
- 再実行率
- 工程別消費比率

## 9. 運用フロー
1. 週初に配分計画を更新する。
2. 日次で `premium-request-usage-log.csv` に実績を追記する。
3. 週末に `pm-weekly-report` へ消費実績と翌週調整案を反映する。
4. 月末に単価・配分・しきい値を見直す。

## 10. 注意事項
- 単価は契約プランにより変動するため、必ず最新契約値で更新する。
- 実績データが取得できない期間は、暫定推計として別タグで管理する。
