# 04 System Context

## システム境界
- 利用者はWeb画面から投稿候補の一覧確認、編集、確認、予約、即時投稿、キャンセルを行う。
- アプリケーションサーバは投稿候補管理、投稿実行、履歴管理、外部連携制御を担当する。
- バッチ処理は情報収集、アノマリー生成、予約時刻到達投稿を担当する。

## コンテキスト

| 区分 | 名称 | 役割 |
|------|------|------|
| 利用者 | 運用担当者 | 投稿候補の確認・編集・承認相当の最終判断 |
| 外部IF | ニュース/イベント/指標配信元 | 投稿候補の材料データを提供 |
| 外部IF | SNS投稿先 | 予約投稿・即時投稿の送信先 |
| 外部IF | 時刻同期基盤 | 予約時刻の基準時刻を提供 |
| 内部機能 | Web UI | 投稿一覧、投稿編集、投稿確認を提供 |
| 内部機能 | API | 画面操作とバッチ処理の共通サービスを提供 |
| 内部機能 | Batch | 候補生成と予約実行を自動化 |
| 内部機能 | Log/Audit | 操作履歴と投稿履歴を保存 |

## Web / API / Batch 境界

| 層 | 主責務 | 主な入出力 |
|----|--------|------------|
| Web | 一覧表示、編集入力、確認操作 | ブラウザ、JSON API |
| API | 投稿候補CRUD、投稿予約、即時投稿、履歴参照 | Web UI、Batch、外部投稿アダプタ |
| Batch | データ収集、アノマリー生成、予約実行 | 外部配信元、内部DB、API |

## 設計上の補足
- 投稿先プラットフォームとデータ収集元は未確定のため、外部IFはアダプタ方式とする。
- 投稿承認フローが必要になった場合に備え、投稿状態は拡張可能な状態管理で定義する。
