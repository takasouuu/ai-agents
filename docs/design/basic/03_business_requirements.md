# 03 Business Requirements

## 機能要件

| 要件ID | 要件内容 | 優先度 | 備考 |
|--------|---------|--------|------|
| REQ-004 | エージェント定義ファイル（.md）の管理・実行 | 必須 | 基本設計～統合テスト用の7エージェント |
| REQ-005 | ワークフロー実行エンジン（run_workflow.sh）の実装 | 必須 | ローカル実行、Docker Compose対応 |
| REQ-006 | ログシステム（プロンプト・出力・実行時間） | 必須 | .runtime/logsへの構造化記録 |
| REQ-007 | Redmine WBS同期機能 | 必須 | 質問票・進捗のTriageLog更新 |
| REQ-008 | トレーサビリティマトリックス生成 | 必須 | 要件→設計→実装→テストの双方向参照 |
| REQ-009 | レビュー管理（AI→人1→人2） | 必須 | 指摘ID採番、未解決事項追跡 |
| REQ-010 | セキュリティスキャン統合（Sonar/CxSAST/AppScan/BlackDuck） | 必須 | 品質ゲート判定 |
| REQ-011 | HTML原本/作業コピー分離（assets/html-received vs working） | 必須 | 設計書との同期確保 |
| REQ-012 | PHP実装テンプレート（フレームワーク・ライブラリ） | 推奨 | 社内ライブラリ前提 |
| REQ-013 | 週次報告自動生成（進捗・品質・コスト・リスク） | 推奨 | PM Agent出力 |
