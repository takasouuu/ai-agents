# 14 Operations Monitoring

## 監視対象

| 区分 | 監視項目 | 目的 |
|------|----------|------|
| Web/API | 応答時間、5xx件数、同時失敗件数 | 利用影響の早期検知 |
| Batch | 実行成否、処理件数、再試行回数、遅延時間 | 予約投稿遅延の検知 |
| 外部連携 | 投稿先接続失敗率、収集元取得失敗率 | 連携障害の検知 |
| 履歴 | 操作履歴未記録件数、投稿履歴未反映件数 | 監査漏れ防止 |

## アラート方針
- WARNING: 単発失敗、しきい値超過予兆。運用担当へ通知。
- ERROR: 再試行後も失敗、投稿遅延発生。運用担当と管理者へ通知。
- CRITICAL: 予約投稿停止、履歴記録停止。即時エスカレーション。

## ログ保管
- アプリケーションログ、ジョブログ、操作履歴、投稿履歴を分離保存する。
- 保管期間は最低90日を前提とし、詳細は運用設計で確定する。

## 運用レポート
- 日次でバッチ成否と失敗件数を集計する。
- 週次で投稿件数、キャンセル件数、外部連携失敗率を報告する。
