# 11 Nonfunctional Requirements

## 性能
- 一覧画面の初回表示は3秒以内を目標とする。
- 一般的なAPI応答は2秒以内、即時投稿APIは10秒以内を目標とする。
- 予約投稿実行バッチは対象件数100件までは1分以内に処理開始できること。

## 可用性
- 予約投稿実行バッチは異常終了時に自動再試行する。
- 外部連携障害時でも画面から履歴参照とキャンセル操作は可能とする。

## 利用性
- 対応クライアントは主要モダンブラウザを前提とする。
- 投稿一覧画面から主要操作へ3クリック以内で到達できること。

## 保守性
- 収集元、投稿先、アノマリールールは差し替え可能な設定構造とする。
- ログと履歴から障害調査に必要な操作順序を再現できること。

## 監査性
- 操作履歴は誰が、いつ、何を変更したかを記録する。
- 投稿履歴は送信先、送信結果、外部参照IDを記録する。
