# 02 Scope and Goals

## スコープイン
- 投稿候補の生成、一覧表示、編集、確認、予約投稿、即時投稿、予約キャンセル。
- 投稿候補生成に必要なニュース・イベント・指標収集、および株価アノマリー整理。
- 投稿状態、操作履歴、投稿履歴の管理。
- Web画面、アプリケーションAPI、定期バッチ、監視・ログの基本設計。

## スコープアウト
- 投稿先プラットフォーム固有仕様の最終確定。
- ニュース配信元・イベント配信元の正式契約と接続試験。
- 外部結合テスト、システムテスト、本番移行手順の詳細。

## ゴール
- 要件定義の内容を15本の基本設計書に分解し、Web / API / Batch の境界を明確化する。
- 未確定事項を質問票へ起票し、後続工程での実装判断が可能な状態にする。
- 要件と設計書の対応関係を追跡表で確認できる状態にする。

## 成功条件
| 観点 | 判定基準 |
|------|----------|
| 要件網羅 | 要件定義に記載された機能・画面・非機能・未確定事項が設計書に反映されている |
| 画面設計 | 投稿一覧・投稿編集・投稿確認の3画面の役割と遷移が明確である |
| バッチ設計 | 投稿候補生成と予約実行の定期処理が定義されている |
| 運用設計 | 操作履歴・投稿履歴・監視・アラート方針が定義されている |
| トレーサビリティ | REQ-001〜REQ-015 の対応先が追跡できる |

## 設計上の前提
- 投稿先は抽象化した「SNS投稿アダプタ」として扱い、具体的な接続先は後続で確定する。
- 株価アノマリーの詳細ルールはマスタ管理できる前提で設計する。
- 収集元は複数登録可能とし、ソースごとに有効/無効を切り替えられる前提とする。
