# 06 Screen Transition

## 画面一覧

| Screen ID | 画面名 | 主目的 | 対応要件 |
|-----------|--------|--------|----------|
| SCR-01 | 投稿一覧画面 | 投稿候補の確認、編集開始、予約キャンセル | REQ-008, REQ-009, REQ-010 |
| SCR-02 | 投稿編集画面 | 投稿本文・投稿予定日時の編集 | REQ-005, REQ-011 |
| SCR-03 | 投稿確認画面 | 予約投稿または即時投稿の最終確認 | REQ-004, REQ-006, REQ-012 |

## 基本遷移
- SCR-01 → SCR-02: 一覧で対象投稿を選択し、編集を開始する。
- SCR-02 → SCR-03: 編集内容を保存し、確認画面へ進む。
- SCR-03 → SCR-01: 予約完了、即時投稿完了、または確認キャンセル後に一覧へ戻る。
- SCR-01 → SCR-01: 一覧から予約キャンセルを実行し、状態を再表示する。

## 遷移仕様

| From | To | 操作 | API |
|------|----|------|-----|
| SCR-01 | SCR-02 | 編集ボタン押下 | GET /api/posts/{id} |
| SCR-02 | SCR-03 | 確認へ進む | PUT /api/posts/{id} |
| SCR-03 | SCR-01 | 予約確定 | POST /api/posts/{id}/reserve |
| SCR-03 | SCR-01 | 即時投稿 | POST /api/posts/{id}/publish |
| SCR-01 | SCR-01 | 予約キャンセル | POST /api/posts/{id}/cancel |

## 補足
- 承認フローが追加される場合は、SCR-03の後ろに承認待ち画面または承認ステータス表示を追加する。
