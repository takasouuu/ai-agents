# 12 Security Design

## 認証・認可
- 画面利用者は認証必須とする。
- 権限は少なくとも閲覧者、運用担当者、管理者に分ける。
- 投稿実行と投稿先設定変更は管理者または運用担当者に限定する。

## データ保護
- 投稿先アクセストークンはアプリケーション設定から分離し、秘匿情報として保管する。
- 外部収集元APIキーも同様に秘匿化し、画面表示しない。

## Web対策
- 画面入力はサーバ側でバリデーションし、XSS対策としてエスケープする。
- 更新系APIはCSRF対策を適用する。
- APIアクセスは認証セッションまたは署名付きトークンで保護する。

## 監査
- 投稿の編集、予約、即時投稿、キャンセル、設定変更を監査対象とする。
- セキュリティインシデント調査用に操作履歴と投稿履歴を突合できるようにする。
