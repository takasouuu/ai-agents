# 15 Acceptance Criteria

## 受入条件

| AC ID | 条件 | 対応要件 |
|-------|------|----------|
| AC-001 | 投稿一覧画面で投稿予定日時と投稿状態を確認できる設計になっている | REQ-008, REQ-009, REQ-010 |
| AC-002 | 投稿編集画面で本文と予約時刻を変更できる設計になっている | REQ-005, REQ-011 |
| AC-003 | 投稿確認画面から予約投稿または即時投稿を実行できる設計になっている | REQ-004, REQ-006, REQ-012 |
| AC-004 | 予約キャンセル、操作履歴、投稿履歴の設計が定義されている | REQ-007, REQ-014, REQ-015 |
| AC-005 | 外部情報収集、アノマリー生成、予約投稿実行のバッチ設計が定義されている | REQ-002, REQ-003, REQ-013 |
| AC-006 | 未確定事項が質問票へ整理され、トレーサビリティが更新されている | 全要件 |

## 判定基準
- AC-001〜AC-006 をすべて満たした場合に基本設計完了とする。
- 未確定事項が未解決でも、質問票に論点・影響・確認先が整理されていれば次工程へ引き継ぎ可能とする。
