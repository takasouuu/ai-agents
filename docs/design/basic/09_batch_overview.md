# 09 Batch Overview

## バッチ一覧

| Batch ID | 名称 | 実行タイミング | 主な処理 | 対応要件 |
|----------|------|----------------|----------|----------|
| BAT-01 | 外部情報収集バッチ | 5分〜15分間隔 | ニュース・イベント・指標情報を収集し候補材料を蓄積する | REQ-002 |
| BAT-02 | アノマリー生成バッチ | 日次または市場イベント前 | アノマリー定義ルールから投稿候補を生成する | REQ-003 |
| BAT-03 | 予約投稿実行バッチ | 毎分 | 予約時刻到達の投稿をSNS投稿アダプタへ送信する | REQ-004, REQ-013 |
| BAT-04 | 投稿結果反映バッチ | BAT-03実行後 | 投稿結果を投稿履歴へ反映する | REQ-015 |

## バッチ共通方針
- バッチ起動結果はジョブ履歴に保存する。
- 失敗時は最大3回まで再試行する。
- 再試行失敗時はアラートを発報し、未送信または未生成データを再実行対象として残す。

## 入出力概要

| Batch ID | 入力 | 出力 |
|----------|------|------|
| BAT-01 | 外部配信元API、取得条件 | 候補材料データ |
| BAT-02 | 候補材料データ、アノマリールール | 投稿候補 |
| BAT-03 | 予約済み投稿、投稿先接続情報 | 投稿結果、状態更新 |
| BAT-04 | 外部投稿結果、投稿ID | 投稿履歴、監視メトリクス |
