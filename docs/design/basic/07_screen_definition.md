# 07 Screen Definition

## SCR-01 投稿一覧画面

### 目的
- 投稿候補を一覧で確認し、編集、即時投稿、予約キャンセルを行う。

### 主な表示項目
| 項目 | 内容 |
|------|------|
| 投稿ID | 投稿候補の識別子 |
| 候補種別 | 情報収集由来 / アノマリー由来 |
| 投稿予定日時 | 予約済みの場合の実行予定時刻 |
| 投稿状態 | 下書き / 予約済み / 投稿済み / キャンセル |
| 投稿本文サマリ | 投稿本文の先頭要約 |
| 最終更新者/日時 | 最終操作情報 |

### 操作
- 編集
- 予約キャンセル
- 履歴参照

## SCR-02 投稿編集画面

### 目的
- 投稿本文、投稿予定日時、補足メモを修正する。

### 主な入力項目
| 項目 | 型 | 必須 | バリデーション |
|------|----|------|----------------|
| 投稿本文 | テキストエリア | Yes | 空不可、プラットフォーム文字数上限以内 |
| 投稿予定日時 | 日時 | No | 現在時刻より未来 |
| 根拠情報 | 表示のみ | Yes | 収集情報またはアノマリー根拠を表示 |
| 補足メモ | テキストエリア | No | 500文字以内 |

### 操作
- 保存
- 確認へ進む
- 一覧へ戻る

## SCR-03 投稿確認画面

### 目的
- 投稿内容と送信条件を確認し、予約または即時投稿を確定する。

### 主な表示項目
| 項目 | 内容 |
|------|------|
| 投稿本文 | 実際に送信する本文 |
| 投稿予定日時 | 予約時刻、即時投稿時は即時実行表示 |
| 投稿先 | 投稿アダプタ名 |
| 根拠情報 | 候補生成理由の要約 |

### 操作
- 予約確定
- 即時投稿
- 編集へ戻る

## 共通仕様
- すべての更新操作は操作履歴に記録する。
- 投稿済みデータは編集不可とし、再投稿は新規候補扱いとする。
