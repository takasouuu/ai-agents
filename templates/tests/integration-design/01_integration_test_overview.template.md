# 内部結合テスト設計書 - テスト概要

## 概要
- **テスト範囲**: 複数機能の連携、Web-API-Batch-DB 横断シナリオ
- **対象**: 実装済み全機能の統合動作確認
- **実施環境**: ローカル統合環境（本線に寄せた構成）
- **テスト手法**:
  - UI自動テスト（Playwright）+ API テスト + DB 検証
  - 業務シナリオベース（エンドツーエンド）
  - 正常系・代替処理・エラーハンドリング

## テスト環境
| 構成要素 | 詳細 |
|---------|------|
| Web Server | Apache 2.4（または内蔵サーバ） |
| PHP | 8.2 以上 |
| DB | MySQL 8.0（Docker 初期化） |
| テストツール | Playwright（Chrome/Firefox）、cURL/HTTPClient |
| ログ確認 | Docker logs のリアルタイム監視 |

## テスト観点
| 観点 | 説明 |
|------|------|
| **業務フロー** | 画面遷移が正常か、データが正しく流れるか |
| **データ整合性** | Web 画面→API→DB への書き込み・参照で一貫性があるか |
| **外部連携** | Batch 処理と DB 更新、Batch→メール送信等の連携 |
| **エラーハンドリング** | ネットワークエラー、DB 接続切断でも復旧できるか |
| **セキュリティ** | CSRF/XSS/SQLi 対策が機能しているか |
| **パフォーマンス** | 複数ユーザー同時実行時に遅延・デッドロックなし |

## テストケース例
1. ユーザー登録→ログイン→商品検索→発注→確認メール受信
2. Batch 定時実行→DB データ更新→管理画面で反映確認
3. 注文キャンセル→在庫復元→別ユーザーが再購入可能
4. 同時発注時の在庫競合制御・トランザクション
5. DB 障害復旧後の未処理タスク再実行
